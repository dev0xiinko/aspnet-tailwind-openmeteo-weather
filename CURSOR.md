### Common Cursor workflows

- **Run Tailwind in watch mode**
  - Use the integrated terminal:

    ```bash
    npm run dev:css
    ```

  - Keep this running while you edit `.cshtml` or Tailwind classes.

- **Build CSS once (for publish or a quick test)**

  ```bash
  npm run build:css
  ```

- **Run the backend**

  ```bash
  dotnet run
  ```

### Docker workflows

- **Build the image**

  ```bash
  docker build -t practice1-weather .
  ```

- **Run the container**


### Conventions to keep

- **Styling**
  - Prefer Tailwind utility classes over custom CSS when possible.
  - Keep any global CSS additions inside `wwwroot/css/input.css`, then rebuild Tailwind.

- **Backend**
  - Use `HttpClientFactory` (named clients) instead of `new HttpClient()` manually.
  - Keep API integration logic inside the page model or dedicated services, not in `.cshtml`.
